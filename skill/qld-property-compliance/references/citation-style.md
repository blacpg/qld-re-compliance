# Citation Style

Every compliance assertion cites the rule and the underlying source.

## Format

`<Authority tier> — <Source title> <pinpoint> (rule <rule_id>)`

Examples:

- Legislation — Property Occupations Act 2014 (Qld) s 102 (rule RULE-FORM6-001)
- Regulation — Property Occupations Regulation 2014 (Qld) (rule RULE-COMM-002)
- Regulator guidance — ACCC, false or misleading representations about price (rule RULE-MKTG-003)

## Rules

- Always name the jurisdiction: "(Qld)" or "(Cth)".
- Always give the pinpoint where the rule has one (section, schedule, clause, or form field).
- Pin to a version or date when currency matters: "as at <date in version_or_as_at>".
- Link to the `official_url` from the source register when a link is useful.
- Quote only when the wording is decisive, 25 words maximum, with the pinpoint.

## Never

- Never cite a third-party summary or blog as if it were the law.
- Never cite a `source_id` that is not in `sources/register.yaml`.
- Never present a pinpoint you have not verified as `verified`.
